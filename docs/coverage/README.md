# Coverage Docs

Coverage docs are hand-maintained onboarding artifacts. They describe what each
covered upstream or product-adjacent surface contains, which SDK/dispatch/actions
are wired, and what remains unsupported.

Use these with [../dev/SERVICES.md](../dev/SERVICES.md) and the generated
catalogs in [../generated](../generated/). The generated catalogs are
authoritative for the current runtime surface; this directory only lists coverage
notes that actually exist in the repo.
