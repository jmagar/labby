# Codemode Pauses SQLite Store Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the durable SQLite-backed storage layer for Code Mode pause/resume state (bead `lab-yp0s2.1`) — schema + CRUD/query API only, no broker/dispatch wiring, no MCP surface changes.

**Architecture:** A new module `crates/labby/src/codemode/sqlite_pauses.rs` mirrors the existing ACP persistence pattern in `crates/labby/src/acp/sqlite_persistence.rs` exactly: a single-connection `write_pool` (r2d2, `max_size=1`), a multi-connection `read_pool` (r2d2, `max_size=4`, `query_only=true`), WAL mode, all pragmas applied via `SqliteConnectionManager::with_init`. Two tables — `codemode_runs` (one row per paused/resumed execution) and `codemode_call_log` (one row per call attempt, `(run_id, seq)` primary key) — live in a new file `~/.labby/codemode_pauses.db`, resolved via `labby_runtime::lab_home()` (NOT the hand-rolled `$HOME/.lab` path `acp/sqlite_persistence.rs` uses — that stale path is a known, separate bug; do not copy it). Call-log params are redacted before every write by reusing Code Mode's own `redact_trace_value` (currently `pub(crate)` in `labby-codemode`; Task 1 makes it `pub` and re-exports it so this new module can call it without duplicating redaction rules).

**Tech Stack:** Rust, `rusqlite` 0.39 (bundled SQLite), `r2d2`/`r2d2_sqlite` connection pooling, `thiserror` for the local error type, `ulid` for execution IDs (generated by the caller in a later bead — this bead only stores whatever ID string it's given), `sha2` for params hashing, `tokio::task::spawn_blocking` to keep SQLite calls off the async executor.

## Global Constraints

- This bead delivers **only** the storage layer (schema + CRUD/query API). No broker/dispatch wiring, no MCP surface changes — those are beads `lab-yp0s2.2`/`.3`/`.4`.
- The module MUST live in the `labby` binary crate (`crates/labby/src/codemode/`), NOT in `labby-codemode` (which is documented to stay storage-neutral, host-neutral) and NOT in `labby-apis` (no SQLite drivers there beyond what's already scoped).
- **`labby-codemode` is an *optional* dependency of the `labby` binary crate, gated behind the `gateway` Cargo feature** (`crates/labby/Cargo.toml`: `labby-codemode = { path = "../labby-codemode", optional = true }`, `gateway = ["dep:labby-codemode", "dep:labby-gateway"]`). This new `codemode` module calls `labby_codemode::redact_trace_value`, so the module itself — and its `mod`/`pub mod` declarations in `main.rs`/`lib.rs` — must be gated behind `#[cfg(feature = "gateway")]`, or a narrow build without `gateway` enabled will fail to compile. A Code Mode pause/resume store has no reason to exist in a build with no Code Mode/gateway functionality anyway, so this gating is a correct scope match, not a workaround.
- No `mod.rs` files — `codemode.rs` sibling to `codemode/`, per this repo's module convention.
- DB file: new, separate file `codemode_pauses.db` — do not add tables to the existing `acp.db`.
- Snippet source code is never persisted in this schema — only call metadata/results.
- Every field that could carry secrets (tool call params) must pass through redaction before it touches disk. No raw upstream tool params ever hit the DB file, under any code path — verified by an actual on-disk byte-scan test, not just code review.
- DB file created at mode `0600`. Path resolution must reject any `..` component before the file is opened.
- Native `async fn in trait` only if a trait abstraction is introduced (none is required by this plan). Never `#[async_trait]`.
- `cargo clippy --all-features` must stay clean — no new warnings.

#[derive(Debug, thiserror::Error)]
pub enum CodeModePauseStoreError {
    #[error("codemode pauses db path must not contain `..` components: {0}")]
    PathTraversal(String),
    #[error("sqlite error: {0}")]

/// Reject any path containing a `ParentDir` (`..`) component. Absolute paths
/// are otherwise valid — this only guards against traversal, not location.
fn reject_path_traversal(path: &Path) -> Result<(), CodeModePauseStoreError> {
    if path.components().any(|c| matches!(c, Component::ParentDir)) {
        return Err(CodeModePauseStoreError::PathTraversal(
            path.display().to_string(),
        ));
    }
    Ok(())
}

// ── Connection pool ────────────────────────────────────────────────────────

fn codemode_pragma_init(
    query_only: bool,
) -> impl Fn(&mut Connection) -> rusqlite::Result<()> + Send + Sync + 'static {
    move |conn| {
        conn.busy_timeout(std::time::Duration::from_millis(5_000))?;
        conn.pragma_update(None, "foreign_keys", "ON")?;
        conn.pragma_update(None, "journal_mode", "WAL")?;
        conn.pragma_update(None, "synchronous", "NORMAL")?;
        if query_only {
            conn.pragma_update(None, "query_only", "true")?;
        }
        Ok(())
    }
}

#[cfg(unix)]
fn create_db_file_0600(path: &Path) {
    use std::os::unix::fs::OpenOptionsExt;
    // Only sets mode on creation; if the file already exists, leave perms alone.
    std::fs::OpenOptions::new()
        .write(true)
        .create_new(true) // fails if the file already exists — that's fine
        .mode(0o600)
        .open(path)
        .ok();
}

#[cfg(not(unix))]
fn create_db_file_0600(_path: &Path) {}

/// Durable SQLite-backed store for Code Mode pause/resume state.
///
/// Clone is cheap — pools are internally `Arc`-backed by r2d2.
#[derive(Clone)]
pub struct CodeModePauseStore {
    write_pool: Pool<SqliteConnectionManager>,
    read_pool: Pool<SqliteConnectionManager>,
}

impl CodeModePauseStore {
    /// Open (or create) the store at an explicit path. `db_path` must not
    /// contain `..` components. The file is created with mode 0600 on first
    /// open.
    pub async fn open(db_path: PathBuf) -> Result<Self, CodeModePauseStoreError> {
        reject_path_traversal(&db_path)?;

        let (write_pool, read_pool) = tokio::task::spawn_blocking(move || -> Result<_, String> {
            if let Some(parent) = db_path.parent() {
                std::fs::create_dir_all(parent).map_err(|e| format!("create_dir_all: {e}"))?;
            }
            #[cfg(unix)]
            create_db_file_0600(&db_path);

            let write_manager =
                SqliteConnectionManager::file(&db_path).with_init(codemode_pragma_init(false));
            let write_pool = Pool::builder()
                .max_size(1)
                .connection_timeout(std::time::Duration::from_secs(5))
                .build(write_manager)
                .map_err(|e| format!("build write pool: {e}"))?;

            {
                let conn = write_pool
                    .get()
                    .map_err(|e| format!("get write conn: {e}"))?;
                migrate(&conn).map_err(|e| format!("migrate: {e}"))?;
            }

            let read_manager =
                SqliteConnectionManager::file(&db_path).with_init(codemode_pragma_init(true));
            let read_pool = Pool::builder()
                .max_size(4)
                .connection_timeout(std::time::Duration::from_secs(5))
                .build(read_manager)
                .map_err(|e| format!("build read pool: {e}"))?;

            Ok((write_pool, read_pool))
        })
        .await
        .map_err(|e| CodeModePauseStoreError::Internal(format!("db open join: {e}")))?
        .map_err(CodeModePauseStoreError::Sqlite)?;

        Ok(Self {
            write_pool,
            read_pool,
        })
    }

    /// Open using the default path under the current lab home
    /// (`$LABBY_HOME` or `$HOME/.labby`).
    pub async fn from_lab_home() -> Result<Self, CodeModePauseStoreError> {
        Self::open(resolve_pauses_db_path()).await
    }

    async fn blocking_write<T, F>(&self, label: &'static str, f: F) -> Result<T, CodeModePauseStoreError>
    where
        T: Send + 'static,
        F: FnOnce(&Connection) -> Result<T, rusqlite::Error> + Send + 'static,
    {
        let pool = self.write_pool.clone();
        tokio::task::spawn_blocking(move || {
            let conn = pool
                .get()
                .map_err(|e| CodeModePauseStoreError::Internal(format!("{label} pool get: {e}")))?;
            f(&conn).map_err(|e| CodeModePauseStoreError::Sqlite(format!("{label}: {e}")))
        })
        .await
        .map_err(|e| CodeModePauseStoreError::Internal(format!("{label} join: {e}")))?
    }

    async fn blocking_read<T, F>(&self, label: &'static str, f: F) -> Result<T, CodeModePauseStoreError>
    where
        T: Send + 'static,
        F: FnOnce(&Connection) -> Result<T, rusqlite::Error> + Send + 'static,
    {
        let pool = self.read_pool.clone();
        tokio::task::spawn_blocking(move || {
            let conn = pool
                .get()
                .map_err(|e| CodeModePauseStoreError::Internal(format!("{label} pool get: {e}")))?;
            f(&conn).map_err(|e| CodeModePauseStoreError::Sqlite(format!("{label}: {e}")))
        })
        .await
        .map_err(|e| CodeModePauseStoreError::Internal(format!("{label} join: {e}")))?
    }
}

    /// `raw_params` on each entry must be the UNREDACTED params — redaction
    /// happens here, once, via `labby_codemode::redact_trace_value`, so there
    /// is exactly one place in this module that touches raw upstream params.
    pub async fn append_call_log(
        &self,
        run_id: &str,
        entries: Vec<NewCallLogEntry>,
    ) -> Result<(), CodeModePauseStoreError> {
        let run_id = run_id.to_owned();
        // Redact + hash BEFORE crossing into the blocking closure — this is
        // pure CPU work, not I/O, so it doesn't need spawn_blocking, and it
        // keeps the closure below free of any raw-params handling.
        const MAX_REDACTED_PARAM_BYTES: usize = 4096;
        let prepared: Vec<(NewCallLogEntry, String, serde_json::Value)> = entries
            .into_iter()
            .map(|entry| {
                let params_hash = {
                    let canonical =
                        serde_json::to_vec(&entry.raw_params).unwrap_or_default();
                    let mut hasher = Sha256::new();
                    hasher.update(&canonical);
                    hex::encode(hasher.finalize())
                };
                let redacted = labby_codemode::redact_trace_value(
                    &entry.raw_params,
                    MAX_REDACTED_PARAM_BYTES,
                );
                (entry, params_hash, redacted)
            })
            .collect();

        self.blocking_write("append_call_log", move |conn| {
            let now = now_ms();
            for (entry, params_hash, redacted_params) in &prepared {
                let redacted_params_json = serde_json::to_string(redacted_params)
                    .map_err(|e| {
                        rusqlite::Error::ToSqlConversionFailure(Box::new(e))
                    })?;
                conn.execute(
                    "INSERT INTO codemode_call_log
                         (run_id, seq, tool_id, params_hash, redacted_params,
                          result_hash, ok, error_kind, applied_at_ms)
                     VALUES (?1, ?2, ?3, ?4, ?5, ?6, ?7, ?8, ?9)",
                    rusqlite::params![
                        run_id,
                        entry.seq,
                        entry.tool_id,
                        params_hash,
                        redacted_params_json,
                        entry.result_hash,
                        i64::from(entry.ok),
                        entry.error_kind,
                        now,
                    ],
                )?;
            }
            Ok(())
        })
